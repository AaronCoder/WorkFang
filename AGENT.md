# AGENT.md

本文件用于规范 AI Coding Agent 在 **WorkFang** 仓库中的默认行为。

## 0. 仓库状态说明

- 当前仓库处于 **从 OpenFang Fork 后的初始状态**。
- 在未明确需求前，应优先保持与上游 OpenFang 的结构兼容，避免不必要的大规模重构。

## 1. 语言与输出规范（强制）

- **对话使用中文**（Agent 与用户沟通统一中文）。
- **文档使用中文**（README、设计文档、说明文档等默认中文）。
- **代码使用英文**（标识符、字符串内容按实际业务需要，但代码语义命名优先英文）。
- **代码注释使用英文**（包括行内注释、Doc 注释、README 中的代码注释示例）。

## 2. 执行流程规范

1. 任何修改前，先阅读并理解相关上下文文件。
2. 优先参考根目录 `README.md`、`CONTRIBUTING.md`、`docs/README.md`。
3. 修改完成后，检查是否需要同步更新相关文档，确保文档与代码一致。

## 3. 技术栈与工具约束

- Rust 相关改动遵循现有 workspace 结构与 lint/format 约束。
- Node 生态若新增脚本或包管理操作，优先使用 `pnpm`。
- Python 生态若新增或维护脚本，使用 `.venv` 虚拟环境并维护 `requirements.txt`。
- 前端开发保持 HTML/CSS/JS 分离，避免 inline script 与 inline style。

## 4. 变更原则

- 小步修改、可验证、可回滚。
- 非用户明确要求时，不主动改动无关文件或进行大规模重命名。
- 若发现与本文件冲突的历史约定，先记录并与用户确认后再调整。

## 5. CI/CD 与提交流程（强制）

Agent 在执行代码任务时，必须遵循以下完整流程：

1. **按需实现变更**
   - 仅修改与当前任务相关的文件。
   - 每次变更保持最小闭环，避免混入无关改动。

2. **本地质量检查**
   - Rust 项目至少执行：
     - `cargo fmt --all`
     - `cargo clippy --workspace --all-targets -- -D warnings`
     - `cargo test --workspace`
     - `cargo build --workspace`
   - 若存在前端/Node 子项目，按需执行其 lint/test/build（使用 `pnpm`）。
   - 若存在 Python 子项目，按需在 `.venv` 中执行对应测试与检查。

3. **提交代码（按需提交）**
   - 只有在当前任务形成可交付结果时才提交。
   - 提交信息应清晰说明目的与范围。
   - 避免将未完成或未验证的中间状态提交到主分支流程。

4. **推送门禁（必须满足）**
   - **测试通过 + 编译通过后，才允许推送到 GitHub 仓库。**
   - 任一检查失败时，禁止推送；需先修复并重新执行检查。

5. **推送后同步**
   - 如变更影响使用方式、配置或行为，需同步更新相关文档。
   - 在汇报中说明：执行了哪些检查、结果如何、是否已推送。
